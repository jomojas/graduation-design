# Agent Instructions: CT to PET Medical Imaging Project

This document provides essential context and technical guidelines for AI agents working on the CT to PET medical imaging research project.

## 1. Project Overview
- **Mission**: Convert CT (Computed Tomography) images to PET (Positron Emission Tomography) images using deep learning (GANs).
- **Stack**: FastAPI (Backend), React/Vite/TypeScript/Ant Design (Frontend), PyTorch (Deep Learning).
- **Structure**:
  - `backend/`: Python service for image processing and model inference.
  - `frontend/`: React SPA for user interface and file upload.

## 2. Backend Development (Python/FastAPI)

### Setup & Execution
- **Package Manager**: `uv` (recommended) or `pip`.
- **Install Dependencies**: `uv sync` or `pip install -e .` (from `backend/`).
- **Run Server**: `python run_server.py` (starts Uvicorn on port 8000).
- **Model Checkpoints**: Located in `backend/checkpoints/`. The default model is `generator.pth`.

### Build, Lint, and Test
- **Linting**: Use `ruff check .` or `flake8`.
- **Formatting**: Use `black .` or `ruff format .`.
- **Type Checking**: Use `mypy .`.
- **Testing**: Use `pytest`.
  - Run all tests: `pytest`
  - Run a specific file: `pytest tests/test_converter.py`
  - Run a specific test: `pytest tests/test_converter.py::test_conversion_success`

### Code Style Guidelines
- **Imports**: 
  - Absolute imports preferred (e.g., `from models.generator import Generator`).
  - Order: Standard library, third-party, local modules.
- **Typing**: Use strict type hints for all function signatures and class members.
- **Naming**: 
  - Classes: `PascalCase`.
  - Functions/Variables: `snake_case`.
  - Constants: `SCREAMING_SNAKE_CASE`.
- **Device Management**: Always check for `torch.cuda.is_available()` and default to `cpu` if unavailable.
- **Error Handling**: 
  - Use FastAPI's `HTTPException` for API-level errors.
  - Wrap PyTorch/Image processing in `try...except` blocks with descriptive logging.

## 3. Frontend Development (React/TypeScript)

### Setup & Execution
- **Package Manager**: `npm`.
- **Install Dependencies**: `npm install`.
- **Development**: `npm run dev` (starts Vite on port 5173).
- **Build**: `npm run build` (compiles TS and builds with Vite).

### Build, Lint, and Test
- **Linting**: `npm run lint` (if configured, otherwise use `eslint src`).
- **Type Checking**: `tsc --noEmit`.
- **Testing**: No test framework currently installed. Recommend adding `vitest` for unit tests.

### Code Style Guidelines
- **Components**: 
  - Use Functional Components with `React.FC`.
  - Prefer Ant Design components for UI consistency.
- **Imports**: 
  - Group imports: React, AntD/Icons, Third-party (axios), Local components, Styles.
- **Naming**:
  - Components/Files: `PascalCase` (e.g., `CTUpload.tsx`).
  - Hooks/Utility functions: `camelCase`.
- **State Management**: Use `useState` and `useEffect`. For complex state, consider React Context or `useReducer`.
- **API Interaction**: Use `axios` for all backend requests. Base URL is typically `http://localhost:8000`.

## 4. Interaction Patterns (FE <-> BE)
- **Endpoint**: `POST /convert`
  - **Input**: `Multipart/form-data` with a `file` field.
  - **Output**: JSON `{ "success": boolean, "pet_image_url": "data:image/png;base64,..." }`.
- **Status Checks**: Frontend should check `GET /` or `GET /status` to verify backend availability and model loading status.

## 5. File System & Persistence
- **Uploads**: Temporary files stored in `backend/uploads/`.
- **Outputs**: Generated images stored in `backend/outputs/` (cleaned up after conversion).
- **Environment**: Use `.env` files for configuration, but never commit them (referenced in `.gitignore`).

## 6. Safety & Security
- **Secrets**: Do not hardcode API keys or sensitive paths.
- **File Validation**: Always validate file types (JPG, PNG, DICOM) and size (max 10MB) on both FE and BE.
- **Resource Limits**: Be mindful of VRAM usage during inference. Use `torch.no_grad()` for all predictions.

## 7. Development Philosophy
- **Modularization**: Keep AI models, business logic (services), and API routes (main) separate.
- **Documentation**: Comment complex mathematical logic in GAN layers or image preprocessing steps.
- **Consistency**: Match existing Ant Design patterns in the frontend and FastAPI patterns in the backend.
